Luma Event Scraper

Scraper automatizado que descubre diariamente eventos nuevos de tech y ai en Luma (lu.ma), sin API oficial, para alimentar dashboards, newsletters o alertas.

Objetivo: 1 corrida diaria, sin intervención manual Stack: Python stdlib + GitHub Actions Estado: v1 completada

Alcance v1

Dentro de alcance

  • Consultar categorías tech y ai
  • Cobertura multi-ciudad (≥ 5 coordenadas configurables)
  • Deduplicación entre corridas por api_id
  • Persistencia: estado visto, CSV maestro, CSV diario de novedades
  • Ejecución programada 1 vez al día (cron / GitHub Actions)
  • Reintentos con backoff ante fallos de red

Fuera de alcance (v1)

  • Scraping de listas de asistentes (requiere sesión)
  • Notificaciones push (Slack / Telegram / email)
  • Enriquecimiento de datos de hosts
  • Filtrado semántico por palabras clave
  • UI / dashboard de visualización
  • Deduplicación semántica (solo por api_id)

Requisitos no funcionales

IDRequisito
RNF-1Sin dependencia de API oficial ni API key
RNF-2Respetuoso con el servidor: pausas aleatorias, sin concurrencia agresiva
RNF-3Resiliencia ante cambios de esquema: log + continuar
RNF-4Portabilidad: Python estándar, sin dependencias externas obligatorias
RNF-5Costo: operable en runner gratuito (GitHub Actions free tier)
RNF-6Auditabilidad: campo first_seen en todo evento nuevo

Riesgos y mitigaciones

RiesgoImpactoMitigación
Endpoint no documentado cambia o deja de responder Alto Logging detallado + revisión periódica del payload crudo
Medidas anti-bot (rate limiting, bloqueo IP) Medio Pausas entre requests, sin concurrencia, backoff
Cambios en Términos de Servicio de Luma Medio-Alto Revisión legal antes de uso comercial; no persistir PII
Sesgo geográfico del endpoint Bajo-Medio Consultar múltiples ciudades; monitorear cobertura
